# Webcam-Based Eye Gaze Estimation Engine

## Engineering Design Document (v1.0)

---

# Project Name

**VisionPoint**

*A lightweight webcam-based gaze estimation engine built using OpenCV, MediaPipe and Mathematical Modeling.*

---

# Objective

The objective of this project is to estimate **where a user is looking on a monitor using only a normal webcam**.

Unlike commercial eye trackers that use infrared cameras and specialized hardware, this system relies entirely on:

* OpenCV
* MediaPipe Face Mesh
* Mathematics
* Calibration
* Machine Learning Regression

The final output will be an estimated point on the screen.

# Why This Is Difficult

The webcam **does not know** where someone is looking.

It only sees

```
Image Pixels
```

The computer must infer

```
Eye Position

↓

Eye Rotation

↓

Viewing Direction

↓

Screen Coordinate
```

This makes gaze estimation a computer vision and geometry problem.

---

# Core Principle

The pupil moves inside the eye.

When the eye rotates,

```
Eye Corners

remain almost fixed

while

Pupil Position changes
```

Instead of measuring the pupil's absolute position,

measure its **relative position** inside the eye.

This removes many scaling issues.

# Phase 5

## Mathematics

Euclidean Distance

```
d = √((x₂-x₁)² + (y₂-y₁)²)
```

Normalize

```

=

current distance

/

total eye width
```

Normalization makes the algorithm independent of image resolution.

# Common Problems

### Head Movement

Prediction shifts.

Solution

Head pose estimation.

---

### Glasses

Reflections hide iris.

Solution

Increase exposure compensation.

---

### Low Light

Poor landmark quality.

Solution

Adaptive histogram equalization.

---

### Blink

Iris disappears.

Solution

Ignore blink frames using Eye Aspect Ratio (EAR).

---

### Webcam Angle

Laptop webcams sit above the screen.

Solution

Calibrate using the same viewing setup each session.

# Success Criteria

The system is considered successful if it can:

* Detect a face reliably.
* Track both eyes at interactive frame rates.
* Locate the iris center consistently.
* Estimate gaze direction after calibration.
* Predict the user's approximate screen location with usable accuracy.
* Maintain stable predictions while the user's head moves slightly.
* Run in real time on a standard laptop webcam without specialized hardware.

## Personal Challenge

Knowing the kinds of projects you've been building recently (AI products, developer tools, and hackathon prototypes), I'd encourage treating this not as "another OpenCV project" but as the start of a **modular gaze estimation engine**.

Build every phase as an independent module with clear inputs and outputs. That will make it much easier to improve individual components later—for example, replacing the regression model with a neural network or adding a more sophisticated head-pose estimator—without rewriting the entire application. This modular approach also makes the project a stronger portfolio piece because it demonstrates software architecture in addition to computer vision.
