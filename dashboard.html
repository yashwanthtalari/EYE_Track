Why We Settled on the Final Dual Architecture

1. Reverting the Interactive App: Native Speed

The interactive client applications (app.py and calibration.py) must run at maximum FPS with zero camera-to-cursor lag. The legacy geometric MLP is ideal for this:

  • Microsecond Inference: Running forward passes on just 14 features takes < 0.2ms, causing zero GUI blocking.
  • Zero CPU Load: Running heavy convolutional feature extractions on every camera frame at 30 FPS can peg client CPU threads; the geometric version remains extremely light.
  • Lighting Independence: Landmark locations are highly robust to shadows and camera exposure, where raw pixels easily degrade.

2. PyTorch CNN + ANN: High Precision Tracking

When high-precision gaze estimation is needed (e.g. for offline analysis or structured tracking tasks), the deep learning model represents a huge step forward:

  • Rich Spatial Learning: The CNN automatically learns features like sub-pixel pupil centers and iris boundaries, which are noisy in standard MediaPipe circles.
  • Significant Error Reduction: Lowered mean gaze error to 62.1 px (median 51.3 px), which is a 60% error reduction from the true geometric baseline.
  • Standard Serialization: Cleanly serialized to standard weight files, running fully headlessly for generic pipelines.
